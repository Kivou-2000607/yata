API keys management

The bot takes the keys it needs from the YATA database. Before pulling a key it systematically check Torn idendity of the concerned discord member. This verification is made by an API call /user/discord/ checking that Torn and discord identities match. To solve the bootstrap problem, a set of master keys are given by the bot admins on setup. These keys are only used for this verification procedure. Thus, with this system, to be a bot admin you have:

  • to be verified on the official discord server
  • to be registered on the YATA website

For some commands, the YATA bot pulls private informations using your key. And some of these commands can be triggered by a tier, displaying private data on potentially public channels. To avoid malicious use of these commands a direct message is sent to you when the data is pulled and displayed.

If your are a bot admin and you reset your key, you just have to logout and log back into YATA with your new key.
