Module Verify  

This module brings a set of features and commands to verify a discord account with a torn account. Verifying means:

  • Giving a @Looter role
  • Potentially giving faction and group of factions roles
  • Changing the discord display name to the Torn name

Command !verify  

Links a discord account with a torn account. If no arguments given it verifies the author of the command. It will never remove any role.

Syntax: !verify or !verify <TornID> or !verify <@DiscordMember>

Scope: Anyone can use it in #verify-id.

{#

Use the master key to verify discord ID.

#}

Command !verifyKey  

Links a discord account with the Torn account corresponding to the API key on all mutual servers between the bot and the member. It will never remove any role. Official discord Torn verifiaction is not required.

Syntax: !verifyKey <APIKey>

Scope: Anyone can use it as DM to the bot.

{#

Use the input key to verify discord ID.

#}

Command !verifyAll  

Loops over all members of the server and apply !verify only to non verified members.

Syntax: !verifyAll

Scope: Server admin can use it in #yata-admin.

{#

Use the master key to verify discord ID.

#}

Command !checkFactions  

Loops over all factions and check if members are still part it. Write a report and remove faction and group of faction roles if force is asked.

Syntax: !checkFactions or !checkFactions force

Scope: Server admin can use it in #yata-admin.

{#

Use the master key to get the list of members.

#}

On member join  

The bot automatically applies !verify to new members of a server. It sends a welcome message and the result of the verification in the system channel.
